Font Properties

Changing the appearance of text

Now, in this lesson, we're going to talk a bit more about the different CSS properties that we can set for fonts so we can change the appearance of the text in our websites.

And we've already seen some of them, for example, the color property which sets the text color, but there's other ones as well, such as the font-weight, font-size & font-family, which is the typeface that we want to use.

Font Size

1px = 1/96th inch; in metric units it's about 0.26mm. It's 0.26mm wide & 0.26mm tall, so it's a small cube that's very, very tiny.

The reason why we're using imperial units is coz a lot of the sizing for websites & for design work comes from the olden days of book printing, book binding & it was very traditional.

There are other ways that we can represent the font size when we're writing CSS.

One common way that you'll see in addition to pixels is using a point.

1pt = 1/72nd inch; slightly bigger than a pixel.

If 1/96 of an inch is 0.26mm, 1/72 of an inch is about 0.35mm, just a tiny bit bigger.

The point is actually often used in many different applications. So when you select a 12-size font in, say Word, you're actually selecting 12 points. So that means when you're using 12 points on your web page, you can expect that when a website is @100%, it will look about the same as a 12 font on your word document.

In addition, there are some other ways of representing font-size, the "em" & the "rem".

1em; 100% of parent. The em is actually not pronounced em, it's pronounced "m", so it's pronounced in the same way that the "m" is in the alphabet. Why is that? Well, 1em is defined as the full width of an m. The letter m is probably one of the widest letters in the English alphabet, & this unit simply represents that entire width. It's basically a relative size.

1rem; 100% of root. It's a relative size, but it's now relative, instead of to the parent, it's relative to the root of your HTML file, which if you remember, the root is usually an html element that encloses everything inside.

The main difference between rem & em is the fact that rem is a consistent way of changing the sizing. So I normally recommend that everybody who develops websites, when you're setting the font size, to use the rem. Coz the way that you have different things embedded in others, then the relationship to the parent can very quickly get very confusing & you don't know whose parent you're referring to, especially when you're working in a separate CSS file.

So, understand how the rem works, remember, it is a scaled size of the root, 1rem is 1Xroot size, 2rem is 2Xroot size, & this is a much easier way to control the different sizes of various elements on your screen & to have more clarity around what's going on.

Font Weight

You can have normal or bold.

It just makes the font look a bit heavier.

But out in the wild you might see some other ways that people use to change the font-weight in addition to the basic keywords.

You can have a relative font-weight which is relative to the parent to have the font lighter or have it bolder.

In addition, you can also use a number from 100 up to 900 to determine how light or how bold you want the font to be.

Font Family

It determines what you want your text to look like.

In the first example, we've got the font-family that is set to Helvetica. This is a particular typeface & the name is called Helvetica, but this is a Mac-specific typeface. So there are many Windows computers that do not own this typeface. That's why very often when we write a font-family, we don't just write the font name, we have a comma, & then we have a backup generic font type. So in this case it's a sans-serif.

What is important though, is having the generic typeface. This means that when you have this set, that Mac users will see this font, but Windows users will see something that's relatively equivalent & it will be automatically displayed for them when it doesn't work.

Now, what if you have a font that has many words in it? For example, "Times New Roman" consists of 3 words. Well, in this case, we actually need the quotation marks around it so that we can tell the CSS file that there are spaces in the name of the font family. So just be careful about that when you're using it. And because "Times New Roman" is a serif-type font, then the backup, the generic type is also a serif.

A sans-serif is simply the type of font where all the edges are pretty much @ right angles.

If you had these little feet or these little decorations @ the edges of your letters, then very often what you're looking @ is a serif-type font.

So that's the main 2 types of different typefaces. And in later lessons, we cover typefaces in more detail.
